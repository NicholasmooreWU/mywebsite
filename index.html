Automated Conflict Intelligence Monitor

Overview

Intelligence analysts face an overwhelming volume of unstructured open-source data (OSINT). My goal was to build an automated pipeline that ingests global news streams, extracts key entities, and scores sentiment to help surface emerging patterns in near real time.

Implementation

I engineered an end-to-end pipeline using Python. The system utilizes NewsAPI for automated collection, spaCy (NLP) for named entity recognition, and a sentiment scoring model, with results stored in SQLite and visualized through an interactive Streamlit dashboard and network graph.

Example Code 
import spacy
from vaderSentiment.vaderSentiment import SentimentIntensityAnalyzer

nlp = spacy.load("en_core_web_sm")
analyzer = SentimentIntensityAnalyzer()

def analyze_article(article):
    """Extract entities and sentiment from a news article."""
    doc = nlp(article['content'])

    # Named Entity Recognition: filter for people, orgs, geo-political entities
    entities = [
        (ent.text, ent.label_)
        for ent in doc.ents
        if ent.label_ in ("PERSON", "ORG", "GPE")
    ]
    # Sentiment score: compound ranges from -1.0 (negative) to +1.0 (positive)
    sentiment = analyzer.polarity_scores(article['content'])['compound']

    return {"entities": entities, "sentiment": sentiment, "source": article['source']}

Technologies: Python, NLP (spaCy), Network Graph Theory (PyVis), SQL, Streamlit, OSINT

GPS Interceptor — Aircraft Surveillance & Anomaly Detection System

Overview

Building a real-time aircraft tracking and anomaly detection system capable of monitoring live flight data and identifying implausible flight patterns, such as sudden position jumps or speeds beyond physical aircraft limits.

Implementation

Developed a full-stack tracking platform that integrates the OpenSky Network API for real-time aircraft tracking within configurable geographic bounds. Implemented a haversine-based distance calculation and a continuous monitoring loop that flags anomalous flight data for review.

Example Code 
import sqlite3
import math

def haversine(lat1, lon1, lat2, lon2):
    """Calculate great-circle distance between two GPS coordinates (km)."""
    R = 6371  # Earth radius in km
    phi1, phi2 = math.radians(lat1), math.radians(lat2)
    dphi      = math.radians(lat2 - lat1)
    dlambda   = math.radians(lon2 - lon1)
    a = math.sin(dphi/2)**2 + math.cos(phi1)*math.cos(phi2)*math.sin(dlambda/2)**2
    return R * 2 * math.atan2(math.sqrt(a), math.sqrt(1 - a))

def detect_anomalies():
    conn   = sqlite3.connect("surveillance_log.db")
    cursor = conn.cursor()
    cursor.execute("""
        SELECT icao24, lat, lon, timestamp
        FROM flight_data
        ORDER BY icao24, timestamp DESC
    """)
    rows = cursor.fetchall()
    for i in range(len(rows) - 1):
        curr, nxt = rows[i], rows[i + 1]
        if curr[0] != nxt[0]: continue
        distance_km = haversine(curr[1], curr[2], nxt[1], nxt[2])
        time_diff   = curr[3] - nxt[3]
        implied_kmh = (distance_km / time_diff) * 3600 if time_diff > 0 else 0
        if implied_kmh > 2000:
            reason = f"IMPLAUSIBLE JUMP: {distance_km:.2f} km in {time_diff}s"
            cursor.execute(
                "UPDATE flight_data SET is_anomaly=1, anomaly_reason=? WHERE icao24=? AND timestamp=?",
                (reason, curr[0], curr[3])
            )
    conn.commit()
    conn.close()

  • Database Architecture: Normalized SQLite schema with indexes for O(log n) lookups on 10,000+ flight records

  • Anomaly Detection: Automated threat identification by comparing implied speeds against physical aircraft limits

Example Code 
import torch.nn as nn

class ThreatClassifierCNN(nn.Module):
    """CNN for image classification — analogous to radiographic/sensor analysis."""

    def __init__(self, num_classes=10):
        super().__init__()
        self.features = nn.Sequential(
            # Block 1: learn edges and low-level textures
            nn.Conv2d(3, 32, kernel_size=3, padding=1),
            nn.BatchNorm2d(32), nn.ReLU(inplace=True),
            nn.MaxPool2d(2, 2), nn.Dropout2d(0.25),

            # Block 2: learn shapes and object parts
            nn.Conv2d(32, 64, kernel_size=3, padding=1),
            nn.BatchNorm2d(64), nn.ReLU(inplace=True),
            nn.MaxPool2d(2, 2), nn.Dropout2d(0.25),

            # Block 3: high-level abstractions (vehicle type, silhouette)
            nn.Conv2d(64, 128, kernel_size=3, padding=1),
            nn.BatchNorm2d(128), nn.ReLU(inplace=True),
            nn.MaxPool2d(2, 2),
        )
        self.classifier = nn.Sequential(
            nn.Flatten(),
            nn.Linear(128 * 4 * 4, 512),
            nn.ReLU(inplace=True),
            nn.Dropout(0.5),
            nn.Linear(512, num_classes),
        )

    def forward(self, x):
        return self.classifier(self.features(x))
Key Features
  • Modular Architecture: Separated model, training loop, and evaluation into distinct modules — production-ready code structure
  • Data Augmentation: Random crops, horizontal flips, and color jitter to reduce overfitting on limited data
  • Learning Rate Scheduling: StepLR scheduler drove a jump from 70.8% → 73.2% accuracy at epoch 8
  • Per-Class Evaluation: Identified that visually similar classes (cat/dog) account for the majority of misclassifications — a useful insight for model improvement
  • Top-5 Confidence Scoring: Inference outputs ranked probabilities, not just a single label — enabling uncertainty-aware decisions
  • Model Checkpointing: Best weights automatically saved across epochs for reproducible results

Technologies: Python, PyTorch, torchvision, CNN, Data Augmentation, Model Checkpointing

Stock Market Data Analysis Engine

Overview

Extracting actionable insights from a massive financial dataset containing 34 million+ rows and 4,500 columns. The sheer volume of data required advanced SQL techniques

Implementation

Developed a suite of optimized SQL queries using Common Table Expressions (CTEs) and Window Functions. Implemented database indexing strategies to keep queries performant at scale, enabling volatility and returns calculations across the full historical dataset.

Example Code 
/* Optimized volatility calculation using Window Functions */
WITH daily_returns AS (
    SELECT
        ticker,
        trade_date,
        close_price,
        LAG(close_price) OVER (PARTITION BY ticker ORDER BY trade_date) AS prev_close
    FROM stock_prices
),
returns_calc AS (
    SELECT
        ticker,
        trade_date,
        (close_price - prev_close) / prev_close AS daily_return
    FROM daily_returns
    WHERE prev_close IS NOT NULL
)
SELECT
    ticker,
    ROUND(STDDEV(daily_return) * SQRT(252), 4) AS annualized_volatility
FROM returns_calc
GROUP BY ticker
ORDER BY annualized_volatility DESC;

Example Code 
use pnet::datalink;
use pnet::packet::ethernet::{EthernetPacket, EtherTypes};
use pnet::packet::tcp::{TcpPacket, TcpFlags};

fn main() {
    let interfaces = datalink::interfaces();
    let interface = interfaces
        .into_iter()
        .find(|iface| iface.is_up() && !iface.is_loopback())
        .expect("No suitable network interface found");

    let (_, mut rx) = match datalink::channel(&interface, Default::default()) {
        Ok(datalink::Channel::Ethernet(tx, rx)) => (tx, rx),
        Err(e) => panic!("Failed to create datalink channel: {}", e),
        _ => panic!("Unexpected channel type"),
    };

    loop {
        match rx.next() {
            Ok(packet) => {
                if let Some(ethernet) = EthernetPacket::new(packet) {
                    handle_ethernet_packet(&ethernet);
                }
            }
            Err(e) => eprintln!("Packet capture error: {}", e),
        }
    }
}

Secure Image Steganography Tool

Overview

Engineered a Python tool utilizing ChaCha20 encryption and Least Significant Bit (LSB) encoding. The system demonstrates low-level binary data manipulation, embedding encrypted text data directly into image pixel values.

Example Code 
from cryptography.hazmat.primitives.ciphers import Cipher, algorithms
from PIL import Image

class Encoder:
    def text_encoder(self, text, key):
        kdf = PBKDF2HMAC(algorithm=hashes.SHA256(), length=32,
                         salt=salt, iterations=100000, backend=default_backend())
        algorithm = algorithms.ChaCha20(kdf.derive(key.encode()), nonce)
        cipher    = Cipher(algorithm, mode=None, backend=default_backend())
        encryptor = cipher.encryptor()
        return encryptor.update(text.encode()) + encryptor.finalize()

    def alter_image(self, img, binary_message):
        """Embed binary data into the LSB of each pixel's RGB channels."""
        pixels = img.load()
        data_index = 0
        for y in range(img.size[1]):
            for x in range(img.size[0]):
                r, g, b = pixels[x, y]
                if data_index < len(binary_message):
                    r = (r & ~1) | int(binary_message[data_index]); data_index += 1
                if data_index < len(binary_message):
                    g = (g & ~1) | int(binary_message[data_index]); data_index += 1
                if data_index < len(binary_message):
                    b = (b & ~1) | int(binary_message[data_index]); data_index += 1
                pixels[x, y] = (r, g, b)
        return img

Simple SCM (Rust Implementation)

Overview

Building a minimal, content-addressed Version Control System (VCS) from scratch in Rust, replicating core Git functionality using only standard libraries and cryptographic hashing.

Implementation

Architected scm, a Rust binary managing file snapshots using a Merkle-like tree structure. Implemented SHA-256 hashing for unique content-addressed IDs on every file and commit object.

Example Code 
fn commit(repo: &Repo, message: &str) -> Result<()> {
    let blobs = stage_files(repo)?;
    let commit_obj = CommitObject {
        tree:      hash_tree(&blobs),
        parent:    read_head(repo).ok(),
        author:    get_author_config(),
        message:   message.to_string(),
        timestamp: chrono::Utc::now().to_rfc3339(),
    };
    let commit_json = serde_json::to_string(&commit_obj)?;
    let commit_hash = sha256_hex(commit_json.as_bytes());
    write_object(repo, &commit_hash, commit_json.as_bytes())?;
    write_head(repo, &commit_hash)?;
    println!("Committed: {}", &commit_hash[..8]);
    Ok(())
}
