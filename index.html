Stock Market Data Analysis Engine

The Challenge

Extracting actionable insights from a massive financial dataset containing 34 million+ rows and 4,500 columns. The sheer volume of data required advanced SQL techniques to avoid timeout errors and generate timely reports.

The Solution

Developed a suite of optimized SQL queries using Common Table Expressions (CTEs) and Window Functions. This approach moved the computational load to the database engine, allowing for efficient calculation of complex metrics like day-over-day volatility directly within the SQL layer without external processing.

Key SQL Implementation 
/* Optimized volatility calculation using Window Functions */
WITH stock_changes AS (
  SELECT 
    date, ticker, close,
    /* LAG() accesses previous row data without a self-join for O(n) efficiency */
    LAG(close) OVER (PARTITION BY ticker ORDER BY date) AS prev_close,
    LAG(searches) OVER (PARTITION BY ticker ORDER BY date) AS prev_searches
  FROM stock_data
)
SELECT 
  ticker, 
  AVG(close - prev_close) AS avg_daily_volatility
FROM stock_changes
WHERE prev_searches > 50
GROUP BY ticker
ORDER BY avg_daily_volatility DESC;

Technologies: SQL (PostgreSQL), Window Functions, Query Optimization, Python

League of Legends Algorithmic Performance Engine

The Challenge

Raw match data provides statistics (Kills, Deaths, CS) but lacks context on player "Playstyle." The goal was to algorithmically classify players into archetypes and determine if recent performance improvements were statistically significant.

The Solution

Built a Flask API that ingests match data via concurrent threading (ThreadPoolExecutor). Implemented Unsupervised Machine Learning (K-Means Clustering) to group players based on feature vectors normalized by game duration. Created a custom scoring algorithm to map mathematical cluster centroids to human-readable archetypes like "Aggressive" or "Vision-Control".

Key ML Implementation

    """Use K-means clustering to identify player playstyle based on stats patterns"""
    
    # 1. Feature Engineering: Normalize stats by game duration
    features = []

        duration = match['game_duration'] / 60
        features.append([
            match['damage'] / duration,  # Damage per minute
            match['cs'] / duration,      # CS per minute
            match['vision'] / duration,  # Vision per minute
            match['kda'],
            (match['kills'] + match['assists']) / max(duration, 1) # KP Rate
        ])
