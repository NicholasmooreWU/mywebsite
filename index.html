Automated Conflict Intelligence Monitor

The Challenge

Intelligence analysts face an overwhelming volume of unstructured open-source data (OSINT). My goal was to build an automated pipeline that ingests global news streams, extracts key entities (actors, organizations), and visualizes the "centers of gravity" in geopolitical conflicts in real-time.

The Solution

I engineered an end-to-end intelligence system using Python. The system utilizes NewsAPI for automated collection, spaCy (NLP) for Named Entity Recognition (NER), and VADER for sentiment analysis. Data is archived in a relational SQLite database to ensure historical persistence. The final product is a Streamlit Dashboard featuring an interactive physics-based network graph that reveals hidden connections between political actors.

Key NLP Implementation 
import spacy
from vaderSentiment.vaderSentiment import SentimentIntensityAnalyzer

nlp = spacy.load("en_core_web_sm")
analyzer = SentimentIntensityAnalyzer()

def analyze_article(article):
    """Extract entities and sentiment from a news article."""
    doc = nlp(article['content'])

    # Named Entity Recognition: filter for people, orgs, geo-political entities
    entities = [
        (ent.text, ent.label_)
        for ent in doc.ents
        if ent.label_ in ("PERSON", "ORG", "GPE")
    ]

    # Sentiment score: compound ranges from -1.0 (negative) to +1.0 (positive)
    sentiment = analyzer.polarity_scores(article['content'])['compound']

    return {"entities": entities, "sentiment": sentiment, "source": article['source']}

Technologies: Python, NLP (spaCy), Network Graph Theory (PyVis), SQL, Streamlit, OSINT

GPS Interceptor — Aircraft Surveillance & Anomaly Detection System

The Challenge

Building a real-time aircraft tracking and threat detection system capable of monitoring live flight data, identifying impossible flight patterns (teleportation, supersonic speeds), and visualizing threats on an interactive dashboard. The system needed to integrate external APIs, persist data efficiently, detect anomalies through geospatial calculations, and provide continuous surveillance with automated refresh cycles.

The Solution

Implemented SQLite database architecture with optimized schema design for storing 30-second interval flight data (position, velocity, heading, anomaly flags). Architected an orchestration loop (Sentinel) that continuously cycles through data collection → anomaly detection → visualization every 30 seconds.

Key Detection Algorithm Implementation 
import sqlite3
import math

def haversine(lat1, lon1, lat2, lon2):
    """Calculate great-circle distance between two GPS coordinates (km)."""
    R = 6371  # Earth radius in km
    phi1, phi2 = math.radians(lat1), math.radians(lat2)
    dphi      = math.radians(lat2 - lat1)
    dlambda   = math.radians(lon2 - lon1)
    a = math.sin(dphi/2)**2 + math.cos(phi1)*math.cos(phi2)*math.sin(dlambda/2)**2
    return R * 2 * math.atan2(math.sqrt(a), math.sqrt(1 - a))

def detect_anomalies():
    conn   = sqlite3.connect("surveillance_log.db")
    cursor = conn.cursor()

    cursor.execute("""
        SELECT icao24, lat, lon, timestamp
        FROM flight_data
        ORDER BY icao24, timestamp DESC
    """)

    rows = cursor.fetchall()
    for i in range(len(rows) - 1):
        curr, nxt = rows[i], rows[i + 1]
        if curr[0] != nxt[0]:  # different aircraft
            continue

        distance_km    = haversine(curr[1], curr[2], nxt[1], nxt[2])
        time_diff      = curr[3] - nxt[3]
        implied_kmh    = (distance_km / time_diff) * 3600 if time_diff > 0 else 0

        # Flag teleportation: implied speed exceeds commercial aircraft limits
        if implied_kmh > 2000:
            reason = f"TELEPORTATION: {distance_km:.2f} km in {time_diff}s"
            cursor.execute(
                "UPDATE flight_data SET is_anomaly=1, anomaly_reason=? WHERE icao24=? AND timestamp=?",
                (reason, curr[0], curr[3])
            )

    conn.commit()
    conn.close()

Key SQL Implementation 
/* Optimized volatility calculation using Window Functions */
WITH daily_returns AS (
    SELECT
        ticker,
        trade_date,
        close_price,
        LAG(close_price) OVER (PARTITION BY ticker ORDER BY trade_date) AS prev_close
    FROM stock_prices
),
returns_calc AS (
    SELECT
        ticker,
        trade_date,
        (close_price - prev_close) / prev_close AS daily_return
    FROM daily_returns
    WHERE prev_close IS NOT NULL
)
SELECT
    ticker,
    ROUND(STDDEV(daily_return) * SQRT(252), 4) AS annualized_volatility
FROM returns_calc
GROUP BY ticker
ORDER BY annualized_volatility DESC;

fn get_tcp_flags(tcp: &TcpPacket) -> String {
    let mut flags = Vec::new();
    let flag_bits = tcp.get_flags();

    // Bitwise flag extraction for TCP handshake analysis
    if flag_bits & TcpFlags::SYN != 0 { flags.push("SYN"); }
    if flag_bits & TcpFlags::ACK != 0 { flags.push("ACK"); }
    if flag_bits & TcpFlags::FIN != 0 { flags.push("FIN"); }
    if flag_bits & TcpFlags::RST != 0 { flags.push("RST"); }

    flags.join(",")
}

Secure Image Steganography Tool

Overview

Engineered a secure Python tool utilizing ChaCha20 encryption and Least Significant Bit (LSB) encoding. The system demonstrates low-level binary data manipulation by embedding encrypted payloads directly into image pixel data, ensuring secure and hidden communication.

Key Security & Systems Implementation 
from cryptography.hazmat.primitives.ciphers import Cipher, algorithms
from PIL import Image

class Encoder:
    def text_encoder(self, text, key):
        # 1. Derive strong key using PBKDF2 & SHA256
        kdf = PBKDF2HMAC(
            algorithm=hashes.SHA256(),
            length=32,
            salt=salt,
            iterations=100000,
            backend=default_backend()
        )

        # 2. Encrypt with ChaCha20 stream cipher
        algorithm  = algorithms.ChaCha20(kdf.derive(key.encode()), nonce)
        cipher     = Cipher(algorithm, mode=None, backend=default_backend())
        encryptor  = cipher.encryptor()
        return encryptor.update(text.encode()) + encryptor.finalize()

    def alter_image(self, img, binary_message):
        """Embed binary data into the LSB of each pixel's RGB channels."""
        pixels     = img.load()
        width, height = img.size
        data_index = 0

        for y in range(height):
            for x in range(width):
                r, g, b = pixels[x, y]

                # Bitwise op: clear LSB (& ~1) then set new bit (| bit)
                if data_index < len(binary_message):
                    r = (r & ~1) | int(binary_message[data_index]); data_index += 1
                if data_index < len(binary_message):
                    g = (g & ~1) | int(binary_message[data_index]); data_index += 1
                if data_index < len(binary_message):
                    b = (b & ~1) | int(binary_message[data_index]); data_index += 1

                pixels[x, y] = (r, g, b)
        return img

Simple SCM (Rust Implementation)

The Challenge

Building a minimal, content-addressed Version Control System (VCS) from scratch in Rust. The goal was to replicate core Git functionality — including repository initialization, staging, and commit history — using only standard libraries and cryptographic hashing for data integrity.

The Solution

Architected scm, a Rust binary that manages file snapshots using a Merkle-like tree structure. Implemented SHA-256 hashing to generate unique IDs for every file blob and commit object, ensuring data integrity. Designed the system to serialize metadata into JSON and store objects on disk, enabling features like commit, log, and revert (rolling back to previous states).

Key Systems Implementation 
use serde::{Deserialize, Serialize};

fn commit(repo: &Repo, message: &str) -> Result<(), Box<dyn std::error::Error>> {
    // Snapshot all staged files into content-addressed blob objects
    let blobs = stage_files(repo)?;

    // Build commit object: tree hash + parent hash + metadata
    let commit_obj = CommitObject {
        tree:    hash_tree(&blobs),
        parent:  read_head(repo).ok(),
        author:  get_author_config(),
        message: message.to_string(),
        timestamp: chrono::Utc::now().to_rfc3339(),
    };

    // Serialize to JSON and hash for content-addressability
    let commit_json = serde_json::to_string(&commit_obj)?;
    let commit_hash = sha256_hex(commit_json.as_bytes());

    // Persist object and advance HEAD
    write_object(repo, &commit_hash, commit_json.as_bytes())?;
    write_head(repo, &commit_hash)?;

    println!("Committed: {}", &commit_hash[..8]);
    Ok(())
}
